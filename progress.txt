## 核心亮点总结

### 性能优化栈
```
SQL查询 → 查询缓存 → 列剪枝 → 文件剪枝 → 谓词下推 → 混合查询 → 文件缓存 → DuckDB执行
```
- 查询性能提升2-3倍
- 数据可见延迟降至1-3秒
- COUNT DISTINCT提升10-100倍

### 数据写入流程
```
数据 → WAL → 内存缓冲 → Parquet压缩 → MinIO上传 → 索引更新 → 自动备份 → WAL清理
```
- 端到端数据安全保证
- 批量写入提升吞吐量
- 节点崩溃可恢复

### 技术栈
- 编程语言: Go 1.24+
- 存储: MinIO (S3) + DuckDB (OLAP) + Redis (元数据)
- 格式: Apache Parquet (列式存储)
- 通信: gRPC + RESTful API + Swagger
- 监控: Prometheus (60+指标) + Grafana
- 日志: Zap (结构化日志)

### 部署模式
- **单节点**: 无需Redis，适合开发测试，资源占用少
- **分布式**: 支持水平扩展，高可用，适合生产环境
- **一键切换**: 配置`redis.enabled`即可切换模式，API完全兼容
